We are building AutoFolio as a standalone app within a larger Automotive Apps ecosystem.

Context:
- AutoFolio is separate from SpecHUB
- SpecHUB is the master reference database for vehicle specifications, capacities, part numbers, and other technical data
- AutoFolio must not duplicate or own master global vehicle specification data as its source of truth
- AutoFolio should reference SpecHUB using specID as the long-term linking strategy
- AutoFolio is the dynamic, user-owned, time-based vehicle lifecycle platform

Your role:
Act as the main AutoFolio build architect and implementation planner for this folder/workspace.

What AutoFolio needs to become:
- a powerful but easy-to-use vehicle ownership and service tracking app
- a central place for users to manage their vehicles
- reminder-driven, history-rich, and future-proof
- capable of later supporting workshop/customer interaction

Core data areas to support:
- user vehicles
- vehicle identity and linking to SpecHUB
- registration tracking
- insurance tracking
- service history
- repairs and work history
- invoices and documents
- parts used
- costs
- reminders
- dashboard summary of what is due and important

Future-aware areas to keep in mind:
- workshop business accounts
- workshop/customer linking
- booking and service coordination
- intelligent reminders based on SpecHUB-linked vehicle data

Constraints:
1. Do NOT redesign SpecHUB
2. Do NOT make AutoFolio the source of truth for global vehicle specs
3. Keep ownership data, historical records, and reference data clearly separated
4. Prioritize a clean, scalable, practical architecture
5. Avoid over-engineering the MVP

Your tasks in order:
1. Propose the foundational architecture for AutoFolio
2. Define the core entity model
3. Explain parent-child relationships
4. Recommend how user vehicles should link to SpecHUB
5. Separate MVP schema from future expansion schema
6. Recommend backend and frontend structure for the app
7. Recommend folder/file responsibilities for this project workspace
8. Identify the biggest risks to avoid early

Required output:
- app architecture overview
- entity model
- schema recommendations
- stack recommendation
- project structure recommendation
- MVP plan
- future expansion plan

Important:
This is the main build workspace for AutoFolio, so produce structured output that can be turned directly into implementation files and tasks.
